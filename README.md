## Project Overview
This project proposes an intelligent beauty product recommendation system that seamlessly integrates Computer Vision (CNN), Sequential Modeling (LSTM), and Large Language Models (LLMs). 

By overcoming the cold-start and sparsity limitations of traditional ID-based recommendation systems, our architecture operates entirely within a continuous visual representation space. The system predicts a user's latent aesthetic preference, retrieves the optimal products, and generates personalized, grounded natural-language justifications for the recommendations via a Streamlit web interface.

## Core Architecture

Our pipeline consists of three highly decoupled core modules:

1. ** CNN-based Visual Feature Extraction:** * Utilizes a pre-trained **ResNet50** to extract 2048-dimensional high-level visual features from product images, transforming abstract aesthetics (style, packaging, color) into computable vectors.
2. ** Attention-Enhanced Vector Retrieval LSTM:** * **Pure Representation Learning:** Discards the traditional Softmax classification bottleneck. The LSTM learns the temporal evolution of visual preferences and outputs a single "ideal" aesthetic vector using Mean Squared Error (MSE) loss.
   * **Dynamic Inference:** Employs K-Nearest Neighbors (KNN) cosine similarity for full-matrix vector retrieval, augmented by an auto-regressive Dynamic Purchase Simulation and Stochastic Rejection Mechanism to ensure recommendation diversity.
3. ** LLM-based Grounded Generation (Gemini):** * Replaces generic outputs with personalized recommendation copy generated by **Gemini 2.5 Flash**.
   * Strictly constrained by prompt engineering and metadata grounding to prevent hallucinations. Includes a robust validation loop to filter out repetitive or overly similar texts before displaying them on the UI.

##  Dataset
We utilized the **Amazon Public Dataset (Beauty Category, merged 2018 & 2023)**. 
The raw dataset was rigorously filtered to remove:
* Missing images (essential for CNN)
* Low-activity users (essential for LSTM sequence quality)
* Missing semantic metadata (essential for LLM generation)
